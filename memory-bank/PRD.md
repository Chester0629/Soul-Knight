# PRD.md — 產品需求文件
> Soul Knight (OOP 2025 期末專案)
> 版本: 1.4 | 最後更新: 2026-03-21（整合 Q&A：子彈方向改 WASD、地城 5×5 網格布局、掉落物數值、HUD 素材、掉落物撿取介面）

## 一、專案背景

- **課程**: OOP 2025 期末專案 (OOP2025f_HW0)
- **受眾**: 老師與助教（評分）
- **目標**: 參考 Soul Knight 手遊，實作一個 Roguelike 地城探索遊戲
- **開發方式**: 里程碑優先 (Milestone-driven)，一步一驗收
- **評分標準（已確認）**:
  | 完成範圍 | 分數目標 |
  |---------|---------|
  | M1 + M2（移動、射擊、碰撞、敵人 AI、HUD） | 60 分（及格線） |
  | M3 + M4（多房間地城、Boss 戰、層數切換） | 80 分 |
  | M5（主選單、選角、音效、UI 完整） | 100 分 |

## 二、核心遊戲規則

### 2.1 遊戲結構
- **模式**: 單人 Roguelike（死亡後完整重開，不保留進度）
- **目標**: 通關「第一大關」= **4 層普通層 + 1 層 Boss 層**（共 5 層）
- **進程**: 玩家清空一層所有房間後，使用傳送陣進入下一層；通過第 4 層後進入 Boss 層

### 2.2 房間系統
| 規格 | 說明 |
|------|------|
| Tile 原圖大小 | 16×16 px |
| Tile 渲染大小 | 48×48 px（scale = {3,3}） |
| 測試房間大小（M1） | 初始房間：17×17 格（含牆）= 地板 15×15 + 四面各 1 格牆 |
| 正式房間大小（M3+） | 怪物房間：總格 22×22 ~ 32×32（地板 20×20 ~ 30×30）|
| 每層房間數量 | 6–9 個房間（樹狀/網狀分支結構） |
| 地圖結構 | 每層一張大地圖，多個房間以走廊銜接 |
| 地圖風格 | 1 種（石城風格，f101/w001 系列素材） |
| 門的生成 | 有相鄰房間的方向才開門；敵人全清後才可通行 |
| 下層傳送 | 房間中央出現傳送陣（effect04_5-12.png），踩上觸發進入下一層 |
| 迷你地圖 | 顯示於畫面右中上角，進入房間後才顯示相鄰房間 |
| 視角 | 跟隨玩家捲動（相機系統，即時追蹤，M1 不限制邊界） |
| 房間連接方式 | 有走廊銜接相鄰房間 |

### 2.2.1 地城布局規範（已確認）

所有房間分布在以**初始房間**為中心的 **5×5 網格**（最多 25 格位置）：

```
□ □ □ □ □
□ □ □ □ □
□ □ 初 □ □
□ □ □ □ □
□ □ □ □ □
```

**普通層（第 1~4 層）連接規則**：
- 初始房間 → 2 個基礎小怪房間 → 傳送門房間（玩家必須依序經過）
- 初始房間與傳送門房間各只與 **1 個**基礎小怪房間相連
- 寶箱房間、特殊房間、額外小怪房間隨機生成於基礎小怪房間**相鄰位置**
- 上述輔助房間**必定與至少 1 個基礎小怪房間連接**
- 每個基礎小怪房間最多連接 **1 個**額外小怪房間

**Boss 層（第 5 層）**：
- 結構相同，但傳送門房間改為 **Boss 房間**
- 初始房間 → **3 個**基礎小怪房間 → Boss 房間（比普通層多一個基礎房間）

**房間類型一覽**：
| 房間類型 | 說明 |
|---------|------|
| 初始房間 | 無敵人，玩家出生點 |
| 基礎小怪房間 | 15~25 隻哥布林，分 2~3 波，隨層數增加 |
| 額外小怪房間 | 同基礎小怪房間，隨機出現 |
| 寶箱房間 | 含寶箱（固定掉落武器），無敵人 |
| 特殊房間 | 待 M5 設計 |
| 傳送門房間 | 含傳送陣，清空後啟動 |
| Boss 房間 | 僅含 Boss，第 5 層專用 |

**精英哥布林生成機率**：依層數線性增加（具體數值 M3 實作時確認）

### 2.3 牆壁 Tile 規格（重要）
- **外圍牆壁**：厚度 1 格
- **頂部/底部牆壁**：有上方圖案（例如 w001）+ 側邊圖案（例如 w004）
- **所有牆壁**：全部使用同一類 Tile，不細分方向（以素材視覺區分即可）

### 2.4 角色系統
- **可選角色**: 3 種
  | 角色 | 特性 |
  |------|------|
  | 角色 1（c01） | 可開技能雙持武器 |
  | 角色 2（c02） | 可以翻滾 |
  | 角色 3（c03） | 可施放法陣，陣內可回血 |
- **血量**: 各角色血量不同（待 M5 決定具體數值）
- **能量系統（已確認）**:
  - `m_MaxEnergy = 200`，遊戲開始時為滿能量
  - 手槍：消耗 0（基礎武器，免費使用）
  - 散彈：消耗 10 / 發
  - 雷射：消耗 2 / 幀（持續按住 J 時每幀扣除）
  - **回復來源**：僅限擊敗敵人/開寶箱掉落的 EnergyPickup（能量球）

### 2.5 控制方式
| 操作 | 按鍵/方式 |
|------|-----------|
| 移動 | WASD（可斜向移動，速度需歸一化） |
| 角色朝向 | 只有左/右兩種（上下移動時維持當前左/右朝向） |
| 攻擊 | **`Util::Keycode::J`（已定案）** — 向「最後一次 WASD 輸入方向」發射（含上/下/斜向，需 Normalize）；記錄 `m_LastMoveDir`，與左右朝向 `m_FacingLeft` 獨立，無輸入時沿用上一次方向 |
| 初始移動速度 | 300.0f pixel/second |

### 2.6 武器系統（已確認）
- **武器數量**: 3 種
- **武器攜帶**: 同一時間只能攜帶 1 把（地上只能有 1 把，撿起時自動替換）
- **初始武器**: 手槍（進入遊戲時預設持有，不需撿取）

| 武器 | 來源素材 | 目標路徑 | 能量消耗 | 子彈速度 | 攻擊力 | 備註 |
|------|---------|---------|---------|---------|--------|------|
| 手槍 (Pistol) | `1.07/weapons_19.png` | `Resources/Weapons/weapon_pistol.png` | 0 / 發 | 700.0f px/s | 10 | 基礎武器，按一次 J 射一發 |
| 散彈 (Shotgun) | `1.07/weapons_18.png` | `Resources/Weapons/weapon_shotgun.png` | 10 / 發 | 600.0f px/s | 8（×3 顆） | 同時發射 3 顆扇形子彈 |
| 雷射 (Laser) | `1.07/weapons2_78.png` | `Resources/Weapons/weapon_laser.png` | 2 / 幀 | 持續型 | 5 / 幀 | 按住 J 持續射出，鬆開即停 |

- **子彈碰撞**: 子彈打到牆壁或敵人即消失（雷射為持續光束，以特效表現，M2 可先用普通子彈代替）

### 2.8 玩家碰撞盒規格（AABB）
- 寬：24 px，高：20 px
- 中心點**往下偏移 10px**（`HIT_OFFSET_Y = -10.0f`，Step 1.5 後依視覺效果微調）

### 2.9 玩家初始生成位置（已確認）
- **M1 測試房間**（17×17 格含牆，地板 15×15，中心為世界原點）：初始座標 **(-300.0f, -100.0f)**（房間左側地板區域，在地板範圍內）
- **M3+ 正式地城**：生成在起始房間的 FloorTile 中央（由 Room::SpawnEnemies 同樣方式隨機選取）

### 2.10 掉落物系統
| 掉落物 | 觸發條件 | 效果 |
|--------|----------|------|
| **HealthPickup**（血量球） | 擊殺敵人或開寶箱後**隨機**掉落 | 回復 **+3 HP** |
| **EnergyPickup**（能量球） | 擊殺敵人或開寶箱後**隨機**掉落 | 回復 **+80 m_Energy** |
| **WeaponPickup**（武器） | 寶箱固定掉落 / 特定敵人死亡掉落 | 撿取後替換當前武器；地板上以與 HUD 相同的武器圖示（原尺寸）顯示 |

> [!NOTE]
> **EnergyPickup 是玩家唯一的能量回復管道。** 散彈/雷射玩家一旦耗盡 m_Energy，本層內將永遠無法再使用能量武器，務必確保掉落率合理。

### 2.11 UI 系統
- **HUD 底圖**: 使用 `UI_15.png`（圖內含血條/護盾條/能量條三條組合）
  - 血條（紅色）：以百分比顯示長度，疊加數字 `當前/最大`
  - 護盾條（灰色）：同上（M1 先保留位置，M5 才實作護盾機制）
  - 能量條（藍色）：以百分比顯示長度，疊加數字 `當前/最大`（最大值 200）
- **武器 UI**: 顯示當前武器圖示
- **迷你地圖**: 顯示已發現的房間（右中上角）

### 2.12 Boss 攻擊設計（初步確認，M4 前可細化）
- **素材**: `boss08_0.png` ~ `boss08_7.png`（8 幀循環動畫）
- **Phase 1（HP > 50%）**: 站在房間中央，向玩家方向發射扇形子彈（5 發，每 2.5 秒一輪）
- **Phase 2（HP ≤ 50%）**: 在 Phase 1 基礎上，增加**快速橫向衝刺**（衝刺後短暫停頓，玩家需閃躲）
- **死亡**: 觸發 Victory 畫面（直接切換，無淡出）；背景圖使用 `Settlement screen.png`（GameOver 亦同）

## 四、非功能需求
- **FPS**: 60 fps（PTSD 引擎固定）
- **解析度**: 1280×720
- **語言**: C++17，嚴格分離 .hpp（宣告）與 .cpp（實作），`#pragma once`，繁體中文註解
- **平台**: Windows（CLion + MinGW 開發）
- **音效**: 需要（PTSD 支援音頻，M4/M5 實作）
